W02 Assignment: Explain Abstraction

Question: What is abstraction and why is it important?

Abstraction in programming is the process of hiding complex details and showing only the essential parts of an object or function. It allows developers to manage complexity by breaking down code into simpler, understandable components. Instead of worrying about how something works internally, we focus on what it does. This makes programs easier to read, write, and maintain.

One major benefit of abstraction is code reusability. Once we write a function or class to perform a task, we can reuse it anywhere without rewriting the same logic. It also helps reduce errors because the logic is written once and tested.

A real-world example of abstraction is a car. You press the brake pedal to stop, but you don’t need to know how the hydraulic system works. Similarly, in code, we call a function without knowing all its internal steps.

In my program, I used a function to calculate the total price of items in a shopping cart: 

def calculate_total(prices):
    return sum(prices)

I used this function like this: 

cart = [12.99, 5.49, 3.50]
total = calculate_total(cart)
print(f"Your total is ${total}")

This function hides the logic of adding the prices. It simplifies the program and keeps the main code clean and readable. That’s the power of abstraction.
