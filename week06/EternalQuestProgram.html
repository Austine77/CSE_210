W06 Assignment: Explain Polymorphism

Polymorphism is a principle in object-oriented programming that allows objects of different types to be treated through a common interface. In simple terms, it means using the same method name in different classes but implementing it differently in each class. This enables objects to respond in their own way to the same method call.

Why is it important? One key benefit of polymorphism is that it makes your code more flexible and maintainable. Instead of rewriting or overloading functionality, you can define a general interface in a base class and override it in subclasses. This reduces code duplication and allows you to add new features without modifying existing code.

Example from my code: In my Week 06 assignment, I created a base class called Shape with a method get_area(). I then created subclasses like Rectangle and Circle, each with their own implementation of get_area(). Here's the code example:

class Shape:
    def get_area(self):
        # This is a generic method to be overridden
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def get_area(self):
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def get_area(self):
        return 3.14 * self.radius * self.radius

# Using polymorphism
shapes = [Rectangle(4, 5), Circle(3)]

for shape in shapes:
    print(shape.get_area())

Even though each shape has a different formula for calculating area, I was able to loop through a list of shapes and call get_area() without knowing which specific shape it was. This is the power of polymorphism — writing generalized code that adapts to specific behavior automatically. It’s a key concept in designing clean, scalable applications.
