W06 Assignment: Explain Polymorphism

Polymorphism is a core concept in object-oriented programming that allows objects of different classes to be treated as objects of a common superclass. It means that the same method name can behave differently depending on the object that is calling it. This makes code more flexible and easier to maintain.

A major benefit of polymorphism is that it allows you to write code that works on the interface or base class, while letting the actual behavior be defined in the child classes. This way, you can add new features or classes without changing the existing code structure, which follows the Open/Closed Principle (open for extension, closed for modification).

In my Week 06 program, I created a base class called Shape, which had a method called get_area(). Then, I created different shapes like Rectangle, Circle, and Triangle, all of which inherited from Shape. Each subclass provided its own version of the get_area() method.

Here’s an example of polymorphism in my program:

class Shape:
    def get_area(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
        
    def get_area(self):
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
        
    def get_area(self):
        return 3.14 * self.radius * self.radius

shapes = [Rectangle(4, 5), Circle(3)]

for shape in shapes:
    print(shape.get_area())

Even though each shape has its own get_area() implementation, the loop calls the correct version depending on the object. This shows how polymorphism allows me to write generalized code that still works with specific behavior. It’s a powerful tool that I will definitely use in real-world projects.
